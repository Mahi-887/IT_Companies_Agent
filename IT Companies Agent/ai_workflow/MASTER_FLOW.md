# Master Flow

## Goal

Provide one stable execution path so any AI agent can continue backend work without hesitation.

## Start Sequence

1. Read `HANDOFF_STATE.md`
2. Check `BACKLOG.md`
3. Open the latest daily log
4. Read only the docs needed for the selected task
5. Execute one concrete backend objective

## Work Sequence

1. Confirm repository scope
2. Select the top priority backend item
3. Implement or document the task
4. Verify consistency with existing docs and architecture
5. Update handoff and daily records

## End Sequence

1. Write completed work
2. Write pending work
3. Write blockers if any
4. Write the next starting point
5. Close the session

## Stuck Prevention

- If context is missing, use the latest handoff first
- If a task is too large, reduce it to one session-sized unit
- If implementation cannot proceed, record the blocker clearly
- Never leave the next step undefined
